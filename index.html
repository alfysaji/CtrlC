Networking & Security Basics

Essential fundamental terms and concepts for analyzing, building, and security-auditing computer networks.

TCP

Connection-Oriented
Transmission Control Protocol. A reliable system used to send data between computers safely, making sure everything lands in order.

TCP carries out a "handshake" before sending data. If any information gets damaged or missing while traveling across the network, TCP notices and re-requests it.

Example: Loading this website text uses TCP. If a single piece went missing, words would disappear.

UDP

Connectionless
User Datagram Protocol. A super-fast data delivery system that values pure speed over perfect delivery tracking.

UDP fires information straight to the target without checking if the receiver is ready or if parts are missing. It avoids the heavy delays of double-checking data.

Example: Live video calls or multiplayer video games. A small glitch is ignored to keep the video running in real-time.

IP Address

Layer 3
Internet Protocol Address. A unique numerical identification label assigned to each device operating on a network network.

Just like standard mail delivery services require a home street address, network paths use an IP address to point data toward your laptop or phone.

Example: An identifier look like 192.168.1.50.

MAC Address

Layer 2
Media Access Control Address. A permanent physical ID footprint burned directly onto your hardware network card by manufacturer plants.

While an IP address changes when connecting to new Wi-Fi spots, your MAC address remains identical wherever your device goes.

Example: Looks like 00:1A:2B:3C:4D:5E. Used to block specific devices from access routers.

ARP

Protocol
Address Resolution Protocol. The system that translates a local known IP address into a device's hardware MAC address.

When a router has data meant for an IP address, it shouts across the network: "Who has this IP address?" ARP supplies the matching MAC address response.

Example: Your computer matching 192.168.1.1 to your home router's physical antenna card.

DNS

Application
Domain Name System. The internet phonebook that translates human website text names into computer-readable IP addresses.

Computers don't know what "google.com" means. DNS changes that text name into a numeric IP address instantly behind the scenes.

Example: Typing google.com sends a request to a DNS server, returning 142.250.190.46.

DHCP

Automation
Dynamic Host Configuration Protocol. An automated feature that hands out fresh IP setups to devices joining a network.

Instead of manually typing configuration parameters into every machine, DHCP configures the IP structure for you immediately when you join the network.

Example: Walking into a shop, opening your laptop, and automatically connecting to the guest Wi-Fi.

NAT

Translation
Network Address Translation. A masking method that hides an entire private home network behind one single public facing IP address.

NAT keeps home gadgets hidden from external tracking points by sending all outgoing requests under the router's main profile identity shield.

Example: Your smart TV, phone, and PC all sharing one internet connection to stream movies together.

VPN

Privacy
Virtual Private Network. An encrypted digital tunnel constructed to safely shield traffic traveling over untrusted networks.

A VPN covers data in strong encryption before it hits internet pipes, preventing hackers or ISPs from observing or reading your activity.

Example: Connecting safely to bank servers while working on risky airport public Wi-Fi.

Proxy

Intermediary
An intermediate gateway server standing directly between an active user workstation and target external web platforms.

When using a proxy, your internet requests pass through the proxy server first. The proxy fetches the webpage and passes it back, hiding your identity.

Example: An office blocking social media sites or filtering out hazardous malicious links from employees.

Firewall

Defense
A digital border guard that checks incoming and outgoing network data to block unauthorized access.

Firewalls enforce strict security rules, filtering out bad data packets before they reach your internal systems.

Example: Blocking hacker connection scans while letting normal web searches pass safely.

IDS

Monitoring
Intrusion Detection System. A surveillance sensor that watches network traffic for signs of hacker activity.

Think of an IDS as a security camera. It scans data patterns and sounds an alarm when it spots suspicious movements, but it doesn't stop them itself.

Example: Alerting a security team when a computer tries to guess passwords thousands of times.

IPS

Prevention
Intrusion Prevention System. An active defense tool that finds attacks and blocks them instantly.

Unlike an IDS, an IPS acts like a security guard. When it catches malicious data packets, it drops them right away to protect the network.

Example: Dropping a known malware file attachment from a network stream before it can install.

DMZ

Architecture
Demilitarized Zone. An isolated network buffer zone for public-facing servers, keeping private company data safe behind it.

A DMZ acts like an office lobby. Visitors can access public resources without entering private office backdoors.

Example: Hosting a business website in a DMZ so users can visit without touching confidential files.

Port

Logical Endpoint
A virtual doorway numbers used by operating systems to sort network traffic to the right app.

A single computer handles different tasks at once. Port numbers help route data to the right app, like mail sorted by apartment number.

Example: Web traffic goes through port 80, while secure web traffic uses port 443.

Socket

Software
The combination of an IP address and a Port number to create a single communication channel.

A socket is a programmatic anchor point that lets apps establish active connections across network lines.

Example: Connecting a browser to 192.168.1.1:443 to open a secure configuration panel.

Packet

Data Unit
A small chunk of data sent across a network. Large files are broken into packets before transmission.

Each packet has a header containing the sender IP, receiver IP, and order number, along with the actual payload data.

Example: A digital photo is split into thousands of packets, sent, and rebuilt on your screen.

Gateway

Hardware
A network junction node that converts data between entirely different communication environments.

A gateway acts as an entry point, translating and routing data from one network structure to another.

Example: A home router converting private local traffic into public internet data.

Router

Layer 3 Device
A smart network machine that forwards data packets between different networks to find the fastest path.

Routers act like sorting offices, reading destination IP addresses to pass packets along the best path to their goal.

Example: Your home router taking data from your laptop and passing it out to global servers.

Switch

Layer 2 Device
A hardware controller that hooks up multiple devices within the same local room or office network.

Unlike routers that link networks together, switches direct traffic within a single network, reading physical MAC addresses to talk straight to devices.

Example: Connecting office desktop PCs together via Ethernet cables to share printers.

VLAN

Virtual Network
Virtual Local Area Network. A method to separate a single physical network switch into isolated digital sub-groups.

VLANs organize devices logically, keeping specific network zones separate without needing distinct sets of cables or hardware switches.

Example: Keeping guest Wi-Fi devices from accessing company financial servers on the same switch.

Subnet

Segmentation
Subnetwork. A smaller, distinct segment carved out of a larger IP address block.

Subnetting splits large address blocks into smaller groups to improve speed, manage traffic, and boost security.

Example: Splitting an IP range so HR and Engineering devices stay in separate networks.

CIDR

IP Routing
Classless Inter-Domain Routing. A flexible shorthand notation used to specify network sizes and allocation blocks.

CIDR replaces old, rigid IP classes with a slash notation that tells routers exactly how many IP addresses belong to a network group.

Example: Using 192.168.1.0/24 to mean a network block with 256 available IP slots.

MTU

Measurement
Maximum Transmission Unit. The maximum data packet size (in bytes) that can pass through a network connection.

If a packet is larger than the MTU value, routers split it into smaller pieces (fragmentation), which can slow down speeds.

Example: Most standard internet systems use a default MTU setting of 1500 bytes.

TTL

Packet Lifespan
Time To Live. An expiration counter value on a packet that prevents it from looping forever if it gets lost.

Every router a packet passes through lowers the TTL value by 1. If the counter hits zero, the packet is dropped.

Example: A packet leaves with a TTL of 64, preventing it from looping indefinitely on bad routes.

ICMP

Diagnostics
Internet Control Message Protocol. A system used by network devices to share error reports and diagnostic status data.

ICMP is used to check connection states rather than carry user data payloads like emails or videos.

Example: Running a ping command to test if a web server is online and active.

HTTP

Web Protocol
Hypertext Transfer Protocol. The underlying system used to load and view traditional plaintext website code.

HTTP traffic travels in unencrypted plaintext, meaning anyone monitoring the connection can read the data passing through.

Example: Visiting older informational websites that don't collect passwords or data.

HTTPS

Secure Web
Hypertext Transfer Protocol Secure. An upgraded, encrypted version of HTTP that protects web traffic.

HTTPS secures data with TLS/SSL encryption, making passwords and financial details unreadable to attackers tracking the connection.

Example: Logging into modern banking portals or secure social media sites with a lock icon.

FTP

File Transfer
File Transfer Protocol. An older system designed to copy files from one computer host to another.

FTP sends usernames and passwords across the network in plaintext, making it risky to use on public links.

Example: Moving old design assets to a legacy testing server via port 21.

SFTP

Secure Transfer
SSH File Transfer Protocol. A secure file transfer method that uses SSH encryption to protect data.

SFTP encrypts both login credentials and files, replacing older unencrypted FTP setups.

Example: Uploading new website code files securely to remote web servers.

SSH

Remote Shell
Secure Shell. An encrypted system used to securely log in and run terminal commands on remote computers.

SSH provides strong encryption, letting admins control servers over the internet as if they were sitting right in front of them.

Example: Opening a remote terminal link over port 22 to update a server across the world.

SMTP

Email Sending
Simple Mail Transfer Protocol. The rule engine used to push and route emails from your app to destination mail servers.

SMTP handles the outbound delivery phase of emails, passing messages between mail systems across the web.

Example: Clicking "Send" on an email client to pass data along port 25 or 587.

POP3

Email Fetch
Post Office Protocol version 3. An email retrieval method that downloads messages to your device and deletes them from the server.

POP3 stores your email history locally on one device, meaning you can't view downloaded messages from other computers.

Example: Downloading work emails onto a desktop client to read them offline.

IMAP

Email Sync
Internet Message Access Protocol. A modern email system that syncs messages across multiple devices in real-time.

IMAP leaves emails on the central server, meaning changes like reading or deleting a message update across all your devices.

Example: Reading an email on your phone, and seeing it marked as read on your laptop later.

SNMP

Management
Simple Network Management Protocol. A tool used to monitor and collect status metrics from routers, switches, and servers.

SNMP helps admins track hardware metrics like temperature, CPU use, and link states across a network.

Example: An automated dashboard alerting a tech team if a core router's CPU spikes above 95%.

LDAP

Directory
Lightweight Directory Access Protocol. A central directory system used to look up and check user login profiles within an organization.

LDAP creates a single credential system, letting employees use one login for various office systems and apps.

Example: Logging into an office computer, email profile, and printer using the exact same password.

SMB

File Sharing
Server Message Block. A file-sharing protocol used to access shared assets and printers over a local network.

SMB is commonly used in Windows setups to link office workstations to local shared storage drives.

Example: Opening a local network drive folder to copy spreadsheets over port 445.

NFS

Linux Sharing
Network File System. A file-sharing protocol that lets Linux systems open files over a network as if they were stored locally.

NFS mounts remote server directory trees into a local machine's folder structure across a network connection.

Example: A Linux web node accessing user photo directories stored on a central storage server.

RDP

Remote Desktop
Remote Desktop Protocol. A Microsoft protocol that lets you log in and view a graphical desktop screen of a remote computer.

RDP transmits screen updates, mouse movements, and keyboard inputs, letting you use a remote computer as if you were sitting at it.

Example: Connecting to a remote office desktop from home over port 3389 to run apps.

Telnet

Legacy Link
An older, unencrypted remote terminal login system that has been replaced by secure SSH links.

Telnet sends all data and passwords in clear text, meaning anyone sniffing network traffic can read your login info.

Example: Checking older network hardware configurations using port 23 on private networks.
